# 로컬 실행 가이드 (하이브리드 서치 + Agent)

## 1. 뭘 켜야 하나?

| 서비스 | 용도 | 누가 켜나 |
|--------|------|-----------|
| **MySQL** | DB | 각자 또는 공용 1대 |
| **Qdrant** | 레시피 벡터 DB (dense/sparse) | 각자 또는 공용 1대 |
| **Embedding 서버** (Flask, 5000) | 텍스트 → 벡터 (하이브리드 시 dense 검색용) | **한 명만** 켜도 됨 (아래 참고) |
| **FastAPI** (8001) | OCR, 냉장고 배치(place), **Re-ranking** | Agent에서 Re-ranking 쓸 때만 필요 |
| **Spring Backend** (8080) | 메인 API, Agent | 각자 |


---

## 2. Embedding 서버 — “한 명만 키면 되나?”

**네. 한 명만 켜두면 됩니다.**

- **같은 PC에서 여러 명이 Backend만 돌리는 경우**  
  → 한 명이 Embedding 서버(5000)만 켜두고, 나머지는 Backend만 실행.  
  → `application.yml`에 `app.embedding.url: http://localhost:5000/embed` 이면, **같은 PC**에서는 그 한 명의 5000 포트를 모두가 쓰게 됨 (같은 컴퓨터면 localhost가 같은 인스턴스).

- **각자 다른 PC에서 Backend 돌리는 경우**  
  → **한 명(백엔드 개발자 한 명)**이 Embedding 서버를 자기 PC에서 켜두고,  
  → 나머지는 `app.embedding.url`을 **그 사람 PC IP**로 맞춤.  
  - 예: `http://192.168.0.10:5000/embed` (방화벽/공유기에서 5000 포트 허용 필요).

정리: **Embedding 서버는 팀에서 한 명만 실행해 두고, 나머지는 그 URL만 맞추면 됨.**

## 3. 실행 방법

### 3-1. Embedding 서버 (Flask, 포트 5000)

**한 명이 실행해 두면 됨.**

```bash
cd capstone_backend/python_embedding
pip install -r requirements.txt
python app.py
```

- 첫 실행 시 `jhgan/ko-sroberta-multitask` 모델 다운로드로 시간 걸릴 수 있음.
- 정상이면 `Running on http://0.0.0.0:5000` 로 뜸.
- 다른 PC에서 접속하게 하려면, 실행하는 PC의 IP를 팀원에게 알려주고, 팀원은 `application.yml`에서 `app.embedding.url`을 그 IP:5000으로 설정.

---

### 3-2. FastAPI (포트 8001) — OCR / Place / Re-ranking

**Agent에서 Re-ranking을 쓰거나, 냉장고 OCR·자동 배치를 쓰면 실행.**

```bash
cd capstone_backend/fastapi
pip install -r requirements.txt
uvicorn main:app --host 0.0.0.0 --port 8001
```

- Re-ranking 쓰면 첫 `/rerank` 호출 시 Cross-Encoder 모델 다운로드될 수 있음.
- OCR 쓰려면 `GOOGLE_APPLICATION_CREDENTIALS`, `OPENAI_API_KEY` 등 필요한 env 설정.

---

### 3-3. Qdrant (벡터 DB)

```bash
docker run -d -p 6333:6333 -p 6334:6334 qdrant/qdrant
```

- 레시피 벡터가 이미 들어있는 컬렉션이 있어야 하이브리드/단일 벡터 검색이 동작함.

---

### 3-4. Spring Backend (8080)

```bash
cd capstone_backend/backend
# 환경 변수 설정 후 (OPENAI_API_KEY, DB 등)
./gradlew bootRun
```

- `app.embedding.url`: Embedding 서버 **한 명이 켜둔 주소** (localhost 또는 팀원 IP).
- `app.fastapi.base-url`: Re-ranking/OCR 쓸 때만 FastAPI 주소 (예: `http://localhost:8001`).

---

## 4. 요약

- **하이브리드 서치**: Backend + **Embedding 서버** + Qdrant + MySQL 필요.  
  → **Embedding 서버는 백엔드 개발자 중 한 명만 켜도 되고**, 나머지는 그 URL만 맞추면 됨.
- **FastAPI**: Re-ranking(또는 OCR/place) 쓸 때만 켜면 됨. 실행은 `uvicorn main:app --host 0.0.0.0 --port 8001`.
- **실행 순서**: MySQL, Qdrant → Embedding(한 명) → (선택) FastAPI → 각자 Backend.
